# ماتش‌فلو — MatchFlow

منصة SaaS تجريبية لإدارة المطابقة المالية (Financial Reconciliation) للشركات متعددة الفروع.
مبنية بـ **Next.js 14 (App Router) + TypeScript + Tailwind CSS**، بواجهة خلفية (API Routes)
حقيقية، مصادقة بجلسات موقّعة، وقاعدة بيانات بسيطة (ملف JSON) جاهزة للاستبدال بقاعدة بيانات حقيقية.

## المزايا

- صفحة هبوط كاملة بالعربية (RTL) مع باقات أسعار بالشيكل (₪)
- دخول منفصل تماماً للعملاء (`/login`) وللمسؤولين (`/admin-login` + تحقق ثنائي) — واجهتان مختلفتان بالكامل
- لوحة تحكّم للعميل (`/dashboard`) تعرض عمليات مطابَقة واستثناءات فعلية من الخادم
- لوحة إشراف للمسؤول (`/admin/dashboard`) بعرض شامل لكل الشركات (Multi-Tenant)، محمية بحارس مسارات (`middleware.ts`) يمنع أي مستخدم غير مصادَق كمسؤول من الوصول إلى `/admin/*` ويوجّهه إلى `/admin-login` أو `/access-denied`
- لا يوجد أي رابط ظاهر لصفحة دخول المسؤولين في الصفحة الرئيسية أو شريط التنقل أو الفوتر أو لوحة العميل — الوصول فقط عبر معرفة الرابط مباشرة
- وضع ليلي/نهاري مع حفظ التفضيل
- API حقيقي: تسجيل دخول، تسجيل خروج، جلب/تحديث عمليات المطابقة — كل شيء عبر HTTP فعلي وليس بيانات وهمية في الواجهة فقط

## التشغيل محلياً

يتطلب [Node.js](https://nodejs.org) نسخة 18 أو أحدث.

```bash
npm install
npm run dev
```

ثم افتح المتصفح على http://localhost:3000

## حسابات للتطوير المحلي

هذه الحسابات موجودة مسبقاً في `data/db.json` لأغراض التطوير فقط، ولم تعد مُعبَّأة تلقائياً في نماذج الدخول:

| الدور | البريد | كلمة المرور | ملاحظات |
|---|---|---|---|
| عميل (مستخدم عادي) | `demo@matchflow.io` | `Demo@12345` | من صفحة `/login` |
| مسؤول منصة | `admin@matchflow.io` | `Admin@12345` | من صفحة `/admin-login`، رمز التحقق الثنائي التجريبي: `123456` |

## البنية

```
middleware.ts               # حارس مسارات /admin/* (Edge)، يمنع غير المسؤولين
app/
  page.tsx                 # الصفحة الرئيسية (هبوط) — بدون أي رابط لدخول المسؤولين
  login/page.tsx            # دخول العملاء فقط
  admin-login/page.tsx       # دخول المسؤولين (واجهة منفصلة تماماً + تحقق ثنائي)
  access-denied/page.tsx     # صفحة عرض عند محاولة وصول غير مصرّح بها لمنطقة الإدارة
  dashboard/page.tsx         # لوحة العميل (Server Component محمي)
  admin/page.tsx             # نقطة توجيه: مسؤول → /admin/dashboard، غير ذلك → /admin-login
  admin/dashboard/page.tsx    # لوحة الإشراف الفعلية (Server Component محمي)
  api/
    auth/login/route.ts       # تسجيل دخول العملاء فقط (يرفض حسابات المسؤولين)
    auth/admin-login/route.ts  # تسجيل دخول المسؤولين + OTP
    auth/logout/route.ts
    auth/me/route.ts
    reconciliation/route.ts         # قائمة عمليات المطابقة (محمي بالجلسة)
    reconciliation/[id]/route.ts     # تحديث/حل استثناء يدوياً
components/
  ThemeToggle.tsx, LogoutButton.tsx, ResolveButton.tsx
lib/
  db.ts     # طبقة الوصول لقاعدة البيانات (JSON)
  auth.ts   # تشفير كلمات المرور + جلسات موقّعة (HMAC، بدون مكتبات خارجية)
  plans.ts  # بيانات باقات الأسعار
data/
  db.json   # قاعدة البيانات نفسها (شركات، مستخدمون، عمليات مطابقة)
```

## ملاحظات مهمة قبل الإنتاج

هذا المشروع **سقالة تعليمية/تجريبية (scaffold)** جاهزة للتشغيل والتوسعة، وليست جاهزة للإنتاج كما هي:

1. **قاعدة البيانات**: تُستخدم حالياً ملف `data/db.json` للتبسيط. استبدلها بقاعدة بيانات حقيقية
   (PostgreSQL عبر Prisma مثلاً) قبل أي استخدام فعلي، خصوصاً أن الكتابة على ملف JSON غير آمنة
   عند تعدد الطلبات المتزامنة.
2. **تشفير كلمات المرور**: يُستخدم حالياً `SHA-256` بدون Salt للتبسيط وتفادي أي اعتماديات خارجية
   تحتاج تثبيتاً معقداً. استخدم `bcrypt` أو `argon2` في الإنتاج.
3. **الجلسات**: التوقيع الحالي هو HMAC مخصص بسيط. للإنتاج استخدم مكتبة معتمدة مثل
   `next-auth` أو `iron-session`، وفعّل `SESSION_SECRET` قوي وعشوائي عبر `.env.local`
   (انظر `.env.example`).
4. **رمز التحقق الثنائي (OTP) لصفحة الإدارة**: حالياً رمز ثابت `123456` لأغراض العرض فقط.
   استخدم TOTP حقيقياً (مكتبة مثل `otplib`) مع تطبيق مصادقة مثل Google Authenticator.
5. **عزل المستأجرين (Multi-Tenant)**: العزل حالياً منطقي عبر `tenantId` في طبقة التطبيق.
   لعزل أقوى على مستوى قاعدة البيانات فعلياً، استخدم Row Level Security كما في التصميم الأصلي
   لمنصة ماتش‌فلو (PostgreSQL RLS عبر Supabase مثلاً).
